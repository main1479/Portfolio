<CaseBlock num="01" label="Overview" heading="What it is">
  Cursimax is a **Spanish-language online course marketplace** serving Latin America — 100+ courses, 500+ instructors, and a flow that lets visitors browse, buy, and access courses without a heavy account barrier in the way.

  I built the <span className="accent">entire frontend</span> from a Figma design — landing, course discovery, testimonials, newsletter, footer, and the responsive system tying it together.
</CaseBlock>

<CaseBlock num="02" label="My role" heading="What I did">
  **Sole frontend developer.** The client handed over a Figma file and a backend; everything the user sees on cursimax.com is mine.

  - Translating the Figma comps into production HTML, SCSS, and vanilla JavaScript
  - Building a mobile-first responsive layout that holds together from 320px up
  - Wiring up the interactive bits — navigation, carousels, course cards, the newsletter form
  - Cross-browser QA and accessibility passes before handover
</CaseBlock>

<CaseBlock num="03" label="The brief" heading="What it needed">
  An e-learning marketplace lives or dies on **trust and clarity**. Visitors arrive cold, comparing teachers, prices, and credentials. The frontend had to make the value proposition land in the first scroll.

  The brief leaned hard on three pillars — quality of instruction, accessible pricing, and round-the-clock support — and the build had to surface those without burying the catalogue.
</CaseBlock>

<CaseBlock num="04" label="What I built" heading="The approach">
  No framework. No build complexity that didn't earn its place. Hand-written HTML and SCSS that the client could maintain long after I shipped it.

  - **Mobile-first SCSS** with a small token system for colour, spacing, and type
  - **Responsive layout** built on CSS grid and flexbox — no media-query soup
  - **Vanilla JS** for the interactive surfaces — kept light and dependency-free
  - **Figma-to-code fidelity** — pixel-checked across the breakpoints that matter
  - **Semantic HTML** so the catalogue is crawlable and accessible by default
</CaseBlock>

<CaseBlock num="05" label="Stack" heading="Tools used">
  Picked for longevity and zero ops cost. A static frontend the client can host anywhere and edit without onboarding a build pipeline.

  - HTML
  - SCSS (Dart Sass)
  - Vanilla JavaScript
  - Figma (handoff)
  - Git
</CaseBlock>
