# \_specs

Technical specifications. Each spec breaks down **how** a feature will be built: files to create or edit, components, types, edge cases, accessibility considerations.

Workflow:

1. Write the spec here, file named `<feature>-spec.md` — must match the plan filename.
2. Commit and push.
3. Wait for approval before implementing.
4. If the plan or spec turned out wrong, add a `## Retrospective` section noting what was off, after the feature ships.
5. Once the feature ships and I confirm, move the file to `_specs/done/`.

Pair every spec with a matching `_plans/<feature>-plan.md`.
