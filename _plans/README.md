# \_plans

Non-technical, plain-English plans. Each plan describes **what** is being built and **why**, suitable for a stakeholder to skim and approve.

Workflow:

1. Write the plan here, file named `<feature>-plan.md`.
2. Commit and push.
3. Wait for approval before writing the spec.
4. Once the feature ships and I confirm, move the file to `_plans/done/`.

Pair every plan with a matching `_specs/<feature>-spec.md`.
